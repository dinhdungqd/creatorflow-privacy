Privacy Policy

✅ Core Data Commitment

CreatorFlow does not collect, store, sell, or transmit personal data to external servers.

All extension functionality runs entirely locally in the user's browser. No browsing history, prompts, account credentials, images, videos, or any personal information are ever transmitted to the developer or any third party.

⚠ Third-Party Disclaimer

CreatorFlow is not affiliated with, endorsed by, or connected to Google, xAI (Grok), Meta, CapCut, ByteDance, or any third-party AI platform.

This extension is an independent productivity tool. All product names and trademarks belong to their respective owners.

Section 1
Data Collection Statement

CreatorFlow does not collect any personally identifiable information. Specifically, the extension does not collect, access, store, or transmit:

  • Your name, email address, or account credentials
  • Prompts or text you enter into AI platforms
  • Images or videos you upload or generate
  • Browsing history or visited URLs
  • IP addresses or device identifiers
  • Any form of usage analytics or telemetry
No data from your browser or AI sessions is ever sent to the developer or any external server.
Section 2
Local Storage Only

User settings and preferences are stored locally using Chrome's built-in chrome.storage.local API. This data never leaves your device. Stored data includes only:

  • Last active tool tab (VEO, Grok, Seedance, Meta)
  • Download folder name and file prefix preferences
  • Automation configuration options you have set (e.g. concurrent prompts, delay)
  • UI language preference

This data is stored solely to restore your preferences between sessions. It is never read by the developer and never transmitted anywhere.

Section 3
Permissions Explained

CreatorFlow requests only the permissions necessary for its core functionality. Each permission is explained below:

PermissionWhy It Is Needed
tabs The extension may access browser tab information only to provide user-requested workflow features across supported AI websites — for example, detecting which AI site is active to auto-switch the side panel, or reloading a tab after installation. No tab content or URLs are stored or transmitted.
activeTab Allows the extension to interact with the currently open AI site tab when the user initiates an action. Used solely to automate UI interactions on the active page at the user's request.
storage Saves your local preferences (folder names, settings, last-used mode) in Chrome's local storage. Data never leaves your device.
downloads Enables automatic saving of AI-generated videos and images to your local device when generation completes. Files are saved directly to your Downloads folder — no file content is transmitted to any server.
sidePanel Displays the CreatorFlow control panel in Chrome's built-in side panel UI.
debugger Used exclusively to simulate mouse click events (mousePressed / mouseReleased via Chrome DevTools Protocol) on AI platform UI buttons that intercept and block standard programmatic DOM click events. This is a technical requirement because modern AI platforms built with React/Vue prevent automation via element.click(). The debugger permission is used only to dispatch synthetic input events. No page content, DOM data, network traffic, credentials, screenshots, or any user information is read, recorded, or transmitted through this permission. The debugger is attached only to the active AI site tab, only when the user initiates an automation action, and detached immediately after the click is dispatched.
Section 4
Host Permissions & Supported Sites

The extension is active only on the following specific websites, as declared in the extension manifest:

  • labs.google — Google Flow / VEO video generation
  • grok.com — Grok / xAI image and video generation
  • dreamina.capcut.com and jimeng.jianying.com — Seedance / Dreamina generation
  • meta.ai — Meta AI image and video generation

The extension does not run on any other websites. It cannot access your general browsing activity.

Section 5
No Remote Code Execution

All extension logic, UI components, and configuration data are bundled locally within the extension package. No JavaScript code, configuration, or instructions are fetched from external servers at runtime.

The extension operates fully offline after installation. No network requests are made by the extension itself.
Section 6
No Analytics, Tracking, or Crash Reporting

CreatorFlow contains no analytics libraries, telemetry, crash reporting, or usage tracking of any kind. The developer has no visibility into:

  • How often or when you use the extension
  • Which features you use
  • What prompts or content you generate
  • Any errors or crashes that occur
Section 7
No Third-Party Data Sharing

We do not share, sell, rent, or disclose any user data to third parties — because we do not collect any user data in the first place.

The extension does not integrate with any advertising networks, data brokers, or analytics services.

Section 8
Children's Privacy

This extension is not directed at children under the age of 13. We do not knowingly collect information from children. If you believe a child has provided personal information through this extension, please contact us so we can address it.

Section 9
Changes to This Policy

If this Privacy Policy is updated, the revised version will be published with the extension update on the Chrome Web Store. The "Last Updated" date at the top of this page will reflect any changes.

Continued use of the extension after an update constitutes acceptance of the revised policy.
